# Iteration

The most logical way to move forward is through an iterative approach. This means working with one customer at a time, obtaining their specific configuration. Each customer’s configuration can then be used to develop a dedicated testing harness. Using this harness, Eliot will be able to build a new testing framework, leveraging the classic code base to enable rapid testing—similar to the capabilities of a more modern code base.

Engineering wise this will not be an easy project. But Eliot is a immensely capable engineer.  He will draw not only on his legacy knowledge of the existing code base but also all of the innovations he invented for Iguana X. 

Some basic things need to be addressed like switching to [a modern build system](build.md).
